### **The Dawn of World Wide Web**

The foundation of the modern web was laid back in the 1980s and early 1990s. In 1980, the Internet was being developed progressively, which eventually lead Tim Berners-Lee to propose a new system for sharing information across the internet using hypertext in 1989\. This lead to the creation of the World Wide Web’s core technologies: **HTML, HTTP and URL.** The first every website went live in 1991, which is still available today for people to explore: [http://info.cern.ch](http://info.cern.ch). It was very simple static website mainly consisting of text. Later in 1993, the first browser was invented which changed everything in the tech field: **The Mosaic browser**. It allowed users to view visually HTML content and even allowed users to view images inline with texts, making the WWW very engaging and accessible for the general public. After that, everyone became extremely hyped up with creating their own web browsers, eventually leading to the creation of hundreds of web browsers. 

### **The rise of CSS and JavaScript**

This period, from 1990 to 2000, also saw the rise of server-side technologies like *PHP, Python, MySQL, Apache and many more*, which allowed websites to become more sophisticated by enabling them to generate dynamic content and interact with databases. But still, there was a massive problem with websites, for one: they were static, lacked without and unappealing. For that reason, some genius dudes invented *CSS in 1995 and later modified it in 1996*, which was the method of making websites beautiful and appealing. **Brendan Eich** took the responsibility for tackling interactivity issue, by creating a *brand new JavaScript programming language* which enabled websites to be static by adding external functionality. Astonishingly this language was created in just 10 days, yet it is one of the most influential and powerful programming languages now. Web browsers started adapting to new tech trends and integrating CSS and JavaScript technologies for their software products.

### **Later tech issues and the walk of fame by JavaScript**

The fate of CSS was an ordinary one: several extensions were developed in order to make working with it much more convenient. JavaScript, on the other hand, experienced one of the most complicated developments throughout the history of tech industry.

Initially, in 2000s, developers hated JavaScript for its extreme complication, repeatedly stating that it is very messed up and is not suitable for maintaining large-scale applications. Therefore, an extension called **“JQuery”** was invented in 2006 to fix JavaScript and make the work for developers much easier. Developers loved it.

In 2010, Google developed the first ever JavaScript web framework called **“Angular JS”** to make web development much more constructive and comprehensive, as JQuery was not suitable for the products they were developing. Therefore, large companies started moving to Angular JS.

But, the story did not end here… There were many haters of Angular JS, so Facebook decided to launch its own web framework called **React** in order to replace Angular. It quickly became very popular among developers, and is still the most popular web framework out there. 

In 2014, **Vue JS** was invented to fix some issues related to *Angular JS and React* and become the most practical framework, but it failed, just as any other framework. In 2016, Angular continued this half of fame by developing its second model of Angular to fix all this chaos. *(spoiler: it did not).*

Later, in 2017, **WebAssembly** was invented in Rust Programming Language to fix all this madness. But it did not. In 2019, **Svelte** was invented to replace all the frontend frameworks out there and become the favourite framework of developers. Yet, still, it is the most loved framework out of all, as developers adore its simplicity and performance.

In 2019, *React Hooks* were invented to fix react, while in the following year, *Vue API composition* was invented to fix Vue. In 2020, **Solid JS** invented once again to fix all this madness. This list went on and on, with React Compose, HTMX, Svelte Runes, Ember, Next, Nuxt. *Hell yeah….*

This madness still continues, which is the main reason why developers do not want to get into frontend developer. This ultra-competition cannot guarantee one’s stable workflow as one has to constantly evolve and learn new things to keep up with the trends.

I do not believe that it is going to replace all developers. It will, to some extend, replace some certain professions related to programming. One category of it is frontend development. Advanced AI models can generate comprehensive websites in a matter of seconds, which it takes several days of grinding for a typical developer to build. And that makes me very maad, as I can spend days building something impressive, at least for me, and I encounter some random dude that developers 10x better website than mine by just entering several prompts to **Claude.**
